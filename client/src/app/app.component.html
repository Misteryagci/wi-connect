Présentation du problème

Cette application est fait dans le cadre de projet du module CPA (Conception Pratique de l'Algorithmique) de l'UPMC (Université Pierre et Marie Curie) dont vous pouvez accéder à l'énoncé en cliquant ici. La présentation du problème indiqué dans le document de recherche est comme ceci:

Comme il n'y a pas d'infrastructure fixé ni de gestion centralisée présent dans les réseaux Wi-fi, un ensemble dominant connexe (CDS) d'une graphe qui représente le réseau est largement utilisé comme un backbone virtuel. La construction d'un CDS est NP-difficile. Dans cette papier de recherche, nous proposons un nouveau algorithme glouton, appelé S-MIS, à l'aide d'un Arbre de Steiner qui construit un CDS dans un facteur de 4.8 + ln5 depuis la solution optimale.
Nous introduisons aussi une version distribuée de cet algorithme. Nous prouvons que l'algorithme proposé est mieux que le meilleur performance ratio actuel qui est 6.8. Une simulation est conduite pour comparer S-MIS avec sa variation qui est rS-MIS. La simulation montre que les tailles des CDS générés par S-MIS et rS-MIS sont quasiment le même.

Pour l'implémentation des algorithmes sont fait en langage Go en s'appuyant sur la notion de concurrence, la partie visualisation sont fait en TypeScript avec framework Angular 5, D3.js, et Angular Material. Le code source de projet est hébergé sur Github et complètement open source. Ainsi que les modules contenant les structures des données et les implémentation des algorithmes est Go -getable.
